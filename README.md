## 📋 团队概述

Sterilizer（净化战队）是专门用于**项目净化、审计和重构**的专家团队。使用 **SPARI 工作流**，系统化地整理混乱的项目，建立清晰的文档体系，创建项目单一真相源。

### 核心能力

- 📊 **项目评估**：快速判断项目规模（S/M/L），制定整理策略
- 🧹 **环境净化**：零删除策略，归档杂乱文件，生成可执行脚本
- 🔍 **代码审计**：基于"源码即真理"原则，验证功能实现状态
- 📈 **进度追踪**：提取TODO，分析开发进度，计算完成度
- 📚 **知识重构**：重组文档体系，设计知识库架构
- 🗺️ **导航构建**：生成说明文档.md，确保2次点击触达任何信息

## 🔄 SPARI 工作流

```
┌──────────────────────────────────────────────────────────────────┐
│                      SPARI 项目净化工作流                         │
├──────────────────────────────────────────────────────────────────┤
│  1. Scan    (扫描)     项目评估、规模判断        → Alpha         │
│  2. Purge   (净化)     环境治理、文件归档        → Scrub         │
│  3. Audit   (审计)     源码验证、进度核查        → Probe + Pulse │
│  4. Rebuild (重建)     知识重构、文档体系        → Canvas        │
│  5. Index   (索引)     导航构建、入口统一        → Beacon        │
└──────────────────────────────────────────────────────────────────┘
```

### 阶段说明

#### 1️⃣ Scan（扫描）- Alpha

- 扫描项目结构
- 判断项目规模（S/M/L）
- 识别技术栈
- 制定整理策略

#### 2️⃣ Purge（净化）- Scrub

- 零删除策略（仅归档，不删除）
- 归档杂乱文件
- 生成可执行清理脚本
- 保留关键配置

#### 3️⃣ Audit（审计）- Probe + Pulse

**并行执行**：
- **Probe**：代码审计，功能状态标记，差异分析
- **Pulse**：TODO提取，进度追踪，时间线分析

#### 4️⃣ Rebuild（重建）- Canvas

- 设计 /docs 目录结构
- 重组碎片化信息
- 生成核心文档（架构/API/部署/开发）
- 同步核心信息至说明文档.md

#### 5️⃣ Index（索引）- Beacon

- 生成**说明文档.md**（项目单一真相源）
- 创建导航入口
- 确保2次点击触达任何信息
- 更新 README.md

## 🚀 快速开始

### 1. 安装配置

```bash
# 复制协调器Skill
cp -r skills/sterilizer-coordinator ~/.claude/skills/

# 复制专家成员配置
cp agents/sterilizer-*.md ~/.claude/agents/

# 验证安装
ls ~/.claude/skills/sterilizer-coordinator/
ls ~/.claude/agents/sterilizer-*.md
```

### 2. 使用方式

#### 方式1：完整SPARI流程

```
用户: 我的项目很乱，帮我整理一下
```

协调器会自动执行完整SPARI流程。

#### 方式2：单阶段执行

```
用户: 帮我评估一下项目规模
用户: 帮我清理根目录
用户: 帮我审计代码
```

协调器会触发对应单个专家。

#### 方式3：从中间阶段开始

```
用户: 从审计阶段开始，帮我检查代码和进度
```

协调器会跳过前面的步骤，从Audit阶段开始。

## 📋 核心原则

### 团队协作原则

1. **委托优先**：协调器绝不自己动手，只分析和协调
2. **自然语言触发**：使用 "使用 sterilizer-[expert] 子代理执行 [任务]" 格式
3. **用户优先**：不确定时主动询问，不要猜测
4. **灵活应变**：根据实际情况调整框架步骤
5. **结果导向**：以用户满意为目标，灵活调整流程

### 工作原则

1. **零删除策略**：所有文件仅归档，不删除
2. **源码即真理**：以代码为准，不做假设
3. **2次点击触达**：任何信息都在2次点击内可达
4. **单一真相源**：**说明文档.md** 是项目的唯一管理载体

## 📊 项目规模判断标准

| 规模 | 特征 | 策略建议 |
|------|------|----------|
| **Small** | 扁平化结构，<50文件 | 文档合并，简单归档 |
| **Medium** | 标准 `/src`, `/docs`, `/tests` 分离，50-200文件 | 标准化整理，完整审计 |
| **Large** | 模块化结构，多级目录，>200文件 | 模块化文档库，多级索引 |
