# Sterilizer (净化战队) - 项目审计与重构专家团队

## 概述

**Sterilizer（净化战队）** 是一个专注于项目"外科手术式"整理、审计与重构的专家团队。团队由6位专业领域专家组成，执行环境净化、真伪审计、进度核查、知识库构建和入口统一等核心任务。

### SPARI 工作流

```
┌─────────────────────────────────────────────────────────────────┐
│                     SPARI 项目净化工作流                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. Scan    (扫描阶段)     项目评估、规模判断       → Alpha     │
│         ↓                                                       │
│  2. Purge   (净化阶段)     环境治理、文件归档       → Scrub     │
│         ↓                                                       │
│  3. Audit   (审计阶段)     源码验证、进度核查       → Probe     │
│         ↓                                  + Pulse              │
│  4. Rebuild (重建阶段)     知识重构、文档体系       → Canvas    │
│         ↓                                                       │
│  5. Index   (索引阶段)     导航构建、入口统一       → Beacon    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 协作模式

### 模式 A - 全流程链式协作（默认推荐）

适用于完整项目净化，按SPARI流程顺序执行：

```
用户请求 → 协调器分析
  → Alpha (扫描评估)
  → Scrub (环境净化)
  → Probe + Pulse (深度审计)
  → Canvas (知识重建)
  → Beacon (统一索引)
  → 汇总返回
```

### 模式 B - 单阶段专家

适用于用户只需要某个特定阶段的工作：

```
用户请求 → 协调器分析 → 触发单个专家 → 返回结果
```

### 模式 C - 并行协作

适用于审计阶段，Probe和Pulse可并行工作：

```
用户请求 → Alpha → Scrub
  → ┬→ Probe (代码审计) ─┬→ Canvas → Beacon
    └→ Pulse (进度追踪) ─┘
```

## 核心产出物

| 产出物 | 负责专家 | 说明 |
|--------|----------|------|
| 项目规模评估报告 | Alpha | S/M/L 规模判断及策略建议 |

| 说明文档.md | Beacon | 统一导航入口 |

## 零删除策略

团队遵循**零删除策略**：

- 所有杂乱文件移入 `_TEMP_ARCHIVE/YYYY-MM-DD_Cleanup`
- 绝对不删除任何文件，仅移动
- 关键配置文件（.gitignore, .env.example）保留在根目录

## 质量门控

每个阶段完成后都有明确的质量门控：

| 阶段 | 质量门控 |
|------|----------|
| Scan | 项目规模已判断、策略已制定 |
| Purge | 整理脚本已生成、零删除策略已执行 |
| Audit | 源码已审计、差异已标记、进度已核查 |
| Rebuild | /docs 目录已建立、核心文档已生成 |

## 使用示例

### 启动完整项目净化

```
用户: 我的项目目录很乱，文档也不全，帮我整理一下
协调器: 启动 Sterilizer 净化战队，开始SPARI全流程协作...
  1. 使用 sterilizer-alpha 进行项目扫描评估
  2. 使用 sterilizer-scrub 执行环境净化
  3. 使用 sterilizer-probe 进行代码审计
  4. 使用 sterilizer-pulse 追踪开发进度
  5. 使用 sterilizer-canvas 重构知识库
  6. 使用 sterilizer-beacon 生成统一入口
```

### 单阶段调用

```
用户: 帮我评估一下这个项目的规模
协调器: 使用 sterilizer-alpha 专家进行项目评估...
```

## 核心原则

1. **源码即真理** - 不做假设，只看源码
2. **零删除策略** - 所有文件仅归档，不删除
3. **2次点击触达** - 任何信息都在2次点击内可达
4. **文档同步** - 所有变更同步至「说明文档.md」
5. **可执行产出** - 整理脚本必须可直接运行
